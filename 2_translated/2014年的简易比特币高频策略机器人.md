**The Principle of the Strategy**

The principle is extremely simple and can be understood as an approximation of market-making. At first glance, it might seem too easy to make money; almost anyone could have written it back in 2014 when Bitcoin trading bots were just starting out. Initially, I didn't expect it to be so effective. It emphasizes the importance of translating ideas into practice and seizing opportunities.

Like all high-frequency strategies, this one is based on orderbook analysis. Here's an example of a typical Bitcoin exchange order distribution:
![](https://dn-filebox.qbox.me/0d8ec18c831404d3d1c19e17299c78017abcfd48.png)

The left side shows buy orders with different price levels, and the right side shows sell orders. If someone wants to buy Bitcoin without placing a limit order, they would have to "eat" an existing order. The more orders they place, the higher the chances of matching against large sell orders, which could impact prices temporarily but are unlikely to persist. Conversely, if someone wants to sell, prices might quickly recover due to other buyers wanting to enter the market.

For example, in this scenario, if you directly buy 5 bitcoins at a price around 10377, and then immediately try to sell back 5 bitcoins when the opportunity arises (at 10348), there is a profit margin. The strategy would place an order slightly below 10377, say at 10376.99, while simultaneously placing another order just above 10348, like at 10348.01. If the conditions are met (as in this example), you could profit from the difference between these two prices, which is 28.98.

While it's not guaranteed to be that perfect every time, over a long period and with repeated trading, the probability of making a profit is actually quite high.

Here's an explanation using current strategy parameters:
- The parameter values cannot be used now; they are only provided for illustration.
- It searches upward for cumulative sell orders totaling 8 bitcoins. In this case, that would be at 10377, so the selling price would be this value minus 0.01 (the actual decrement could vary randomly).
- Similarly, it searches downward for cumulative buy orders also totaling 8 bitcoins. Here, that would be at 10348, so the buying price would be 10348.01.
- The difference between these two prices is 28.98. If this exceeds the strategy's preset spread of 1.5, it will place orders at these levels and wait for execution. Otherwise, if the spread is below 1.5, a price that meets or slightly deviates from the current market level (such as adding or subtracting 10) would be used.

**Further Explanations**

1. What do I do when I run out of money or coins?
   - When my funds were low, it was common for me to only place orders on one side. However, this isn't a significant problem. You could incorporate logic for balancing between holding cash and cryptocurrencies, but the process might result in losses. Every execution is a matter of probability; I choose to wait and trade off one side rather than balance, even though it means missing out on the other side's trading opportunities.
2. How is position management handled?
   - Initially, trades were full-scale (all-in). Later, positions were divided into different groups based on varying parameters, ensuring not all orders would be filled at once.
3. Is there a stop-loss mechanism?
   - The strategy has complete buy and sell order logic; I don't think it needs a stop-loss (this can be debated), as the opportunity itself is valuable; stopping could mean missing out.
4. How to modify this for coin profitability?
   - The current parameters are symmetrical, meaning they look 8 bitcoins down on the sell side and up on the buy side. You can unbalance these by increasing the cumulative sell order threshold (e.g., 15) to make selling more difficult, which would allow you to potentially buy back at a lower price if it happens frequently enough.

```javascript
// Function to get the current price based on orderbook depth information
function GetPrice(Type, depth) {
    // _C() is a platform error handling function.
    var amountBids = 0;
    var amountAsks = 0;

    if (Type == "Buy") {
        for (var i = 0; i < 20; i++) {
            amountBids += depth.Bids[i].Amount;
            if (amountBids > floatamountbuy) {
                return depth.Bids[i].Price + 0.01;
            }
        }
    }

    if (Type == "Sell") {
        for (var j = 0; j < 20; j++) {
            amountAsks += depth.Asks[j].Amount;
            if (amountAsks > floatamountsell) {
                return depth.Asks[j].Price - 0.01;
            }
        }
    }

    // If no sufficient order is found, return the first ask price.
    return depth.Asks[0].Price;
}

// Function to cancel all open orders
function CancelPendingOrders() {
    var orders = _C(exchange.GetOrders);
    for (var j = 0; j < orders.length; j++) {
        exchange.CancelOrder(orders[j].Id, orders[j]);
    }
}

// Main trading logic
function onTick() {
    // Get current orderbook depth information
    var depth = _C(exchange.GetDepth);

    var buyPrice = GetPrice("Buy", depth);
    var sellPrice = GetPrice("Sell", depth);

    if ((sellPrice - buyPrice) <= diffprice) {
        buyPrice -= 10;
        sellPrice += 10;
    }

    CancelPendingOrders();

    // Fetch account information
    var account = _C(exchange.GetAccount);
    
    // Calculate the amount of Bitcoin that can be bought
    var amountBuy = _N((account.Balance / buyPrice - 0.1), 2);

    // Calculate the amount of Bitcoin that can be sold (no position limits)
    var amountSell = _N(account.Stocks, 2);

    if (amountSell > 0.02) {
        exchange.Sell(sellPrice, amountSell);
    }
    if (amountBuy > 0.02) {
        exchange.Buy(buyPrice, amountBuy);
    }

    // Sleep before the next trading cycle
    Sleep(sleeptime);
}

// Main function to initialize and run the strategy
function main() {
    while (true) {
        onTick();
    }
}
```
