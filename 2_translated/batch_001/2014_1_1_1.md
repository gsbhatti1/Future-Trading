**Principle of the Strategy**

The strategy is incredibly simple in its core principle, akin to a quasi-high-frequency market-making approach. After reading it, you might feel frustrated thinking that anyone could have written it given how straightforward it is. Initially, I didn't anticipate its effectiveness, realizing that one should quickly turn ideas into practice as there's always the potential for unexpected benefits. In 2014 when Bitcoin robots were just emerging, coming up with a profitable strategy was too easy.

Like all high-frequency strategies, this one relies on order books. Here is an example of a typical order distribution from a Bitcoin exchange:

![](https://dn-filebox.qbox.me/0d8ec18c831404d3d1c19e17299c78017abcfd48.png)

On the left are buy orders showing different price levels and their corresponding quantities. On the right, there are sell orders. If someone wants to buy Bitcoin without placing an order, they would have to eat into existing orders, potentially causing a large number of sells at that level to execute, impacting the price. However, this impact usually doesn't last long as others might want to eat up these asks and push the price back upward. The same logic applies when someone wants to sell Bitcoin.

Using the example order book, if one wanted to directly buy 5 bitcoins, it would result in a trade at 10377. If, in response, someone sells 5 bitcoins, the trade occurs at 10348. This difference is the profit margin. The strategy places orders slightly below 10377, say at 10376.99, while simultaneously buying above 10348, such as at 10348.01. If the scenario occurs, this setup would capture the price difference. While it won't always be perfect, in the long run, the probability of making a profit is astonishingly high.

For an explanation with current parameters, which can no longer be used but serve to illustrate: The strategy searches for the buy price where the cumulative sell orders reach 8 bitcoins (10377), and then places the order slightly below this at 10376.99. Similarly, it finds the sell price where the cumulative buy orders amount to 8 bitcoins (10348) and sets up an order just above that at 10348.01. The difference between these prices is 28.98, which exceeds the preset spread of 1.5; hence it places orders at these levels. If the spread falls below 1.5, another price within a range (e.g., plus or minus 10 from the current bid-ask spread) would be used to place an order for picking up opportunities.

**Further Explanation**

1. What if you run out of money or Bitcoin?
In my case with limited funds, this was quite common; most of the time, I only placed orders on one side but it wasn't a big issue. A coin/balance balancing logic could have been added, but that would introduce potential losses due to the probabilistic nature of each trade. To avoid this, I chose to wait for single-sided trades while missing out on the other side.
2. How is position management handled?
Initially, it was full positions with every buy and sell order; later, different groups were defined based on varying parameters so that not all orders would necessarily execute at once.
3. Is there a stop-loss mechanism in place?
The strategy has complete logic for placing both buy and sell orders, which I believe renders the need for a stop-loss optional (a matter of debate). Still, given the probabilistic nature of trades, taking profits is better than stopping out.
4. How can this be adjusted to make Bitcoin purchases instead of sales?
Currently, the parameters are symmetrical with 8 coins on both sides. An imbalance can be introduced by increasing the number of sell orders (15 coins) which makes it harder to sell but more likely for lower buys. This would result in buying bitcoins rather than selling them; reversing this logic would make profits.

```javascript
function CancelPendingOrders() {
    var orders = _C(exchange.GetOrders);
    for (var j = 0; j < orders.length; j++) {
        exchange.CancelOrder(orders[j].Id, orders[j]);
    }
}

function GetPrice(Type, depth) {
    var amountBids = 0;
    var amountAsks = 0;
    if (Type == "Buy") {
        for (var i = 0; i < 20; i++) {
            amountBids += depth.Bids[i].Amount;
            if (amountBids > floatamountbuy) {
                return depth.Bids[i].Price + 0.01;
            }
        }
    }
    if (Type == "Sell") {
        for (var j = 0; j < 20; j++) {
            amountAsks += depth.Asks[j].Amount;
            if (amountAsks > floatamountsell) {
                return depth.Asks[j].Price - 0.01;
            }
        }
    }
    return depth.Asks[0].Price
}

function onTick() {
    var depth = _C(exchange.GetDepth);
    var buyPrice = GetPrice("Buy", depth);
    var sellPrice = GetPrice("Sell", depth);
    if ((sellPrice - buyPrice) <= diffprice) {
        buyPrice -= 10;
        sellPrice += 10;
    }
    CancelPendingOrders();
    var account = _C(exchange.GetAccount);
    var amountBuy = _N((account.Balance / buyPrice - 0.1), 2);
    var amountSell = _N(account.Stocks, 2);
    if (amountSell > 0.02) {
        exchange.Sell(sellPrice, amountSell);
    }
    if (amountBuy > 0.02) {
        exchange.Buy(buyPrice, amountBuy);
    }
    Sleep(sleeptime);
}
```
