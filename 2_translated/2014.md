**Principle of the Strategy**

The principle behind this strategy is extremely simple, akin to an imitative market-making strategy at lower frequencies. Readers might think that it should be easy to make money this way, as anyone could have written such a strategy during the early days of Bitcoin trading in 2014. I didn't anticipate its effectiveness when first developing it; however, acting on one's ideas can lead to unexpected benefits. Like all high-frequency strategies, this one is based on order book analysis. The following image illustrates a typical order distribution for a Bitcoin exchange:
![](https://dn-filebox.qbox.me/0d8ec18c831404d3d1c19e17299c78017abcfd48.png)
On the left are buy orders, showing quantities at various prices. On the right are sell orders. If someone wants to buy Bitcoin and does not want to place a limit order, they must take an existing offer (taker). If their order is large enough, it could trigger many sell orders, impacting the price. However, such impacts do not usually persist; others will soon try to take offers as well, causing prices to quickly revert. Conversely, if someone wants to sell Bitcoin, a similar dynamic occurs.

For example, suppose one wishes to buy 5 bitcoins directly. In that case, the price would reach 10377. If then, someone sells 5 bitcoins, the new price might drop to 10348, creating a profit margin. The strategy places orders just below 10377 at 10376.99 and buys slightly above 10348 at 10348.01. If such conditions arise, it will earn the difference between these prices. While not perfect every time, this strategy takes advantage of price differences over many transactions to ensure a high chance of profit.

Here's an explanation of the specific operations with current parameters (which can no longer be used). The parameter is simply for illustrative purposes: it searches for the sell order depth where cumulative volume reaches 8 bitcoins. Here, that would be at 10377; thus, this price minus 0.01 becomes the selling price (0.01 can vary randomly). Similarly, find the buy order depth with a cumulative volume of 8 bitcoins, which is here 10348; hence, the buying price becomes 10348.01. The difference between these prices is 28.98, exceeding the strategy's pre-set spread of 1.5, at which point both orders are placed to wait for execution. If the spread is less than 1.5, a different price can be chosen (e.g., adding or subtracting 10 from the current market price) and waiting for an opportunity.

**Further Clarifications**

1. What if there's no money or coins?
In my case with limited capital, it was common to only place one side of the order. It wasn't a significant issue overall. However, incorporating balance logic between coins and money could be done but might result in losses due to the probabilistic nature of each trade. I chose to wait for single-side execution while foregoing the other side's opportunities.
2. How is position managed?
Initially, full positions were taken on both buying and selling; later, trades were divided into different groups based on parameters, preventing complete order fulfillment at once.
3. Are there stop losses?
The strategy has a complete logic for placing orders to buy and sell. I do not believe in using stop losses (this can be debated); additionally, opportunities are seen as chances rather than losses due to the probabilistic nature of trades.
4. How can this be modified to make profits with coins instead of money?
Currently, parameters are symmetrical; adjust them by setting cumulative sell order depth at 15 bitcoins and cumulative buy order depth at 8 bitcoins. This imbalance increases the likelihood of selling low while buying higher. Conversely, this would generate profit in terms of money rather than coins.

Firstly, a function to get the buy and sell prices called `GetPrice()` is defined:
```javascript
function GetPrice(Type, depth) {
    var amountBids = 0;
    var amountAsks = 0;

    // Calculate the buy price where cumulative bid volume reaches the pre-set value.
    if (Type == "Buy") {
        for (var i = 0; i < 20; i++) {
            amountBids += depth.Bids[i].Amount;
            if (amountBids > floatamountbuy) {
                // Add a tiny margin to ensure the order is placed in front.
                return depth.Bids[i].Price + 0.01;
            }
        }
    }

    // Similarly, calculate the sell price.
    if (Type == "Sell") {
        for (var j = 0; j < 20; j++) {
            amountAsks += depth.Asks[j].Amount;
            if (amountAsks > floatamountsell) {
                return depth.Asks[j].Price - 0.01;
            }
        }
    }

    // If no matching order is found, return the first ask price.
    return depth.Asks[0].Price;
}
```

The `onTick()` function then executes these logic:
```javascript
function onTick() {
    var depth = _C(exchange.GetDepth());
    var buyPrice = GetPrice("Buy", depth);
    var sellPrice = GetPrice("Sell", depth);

    // Adjust the spread if it is less than the predefined value.
    if ((sellPrice - buyPrice) <= diffprice) {
        buyPrice -= 10;
        sellPrice += 10;
    }

    CancelPendingOrders();

    // Retrieve account information to determine available balance and holdings.
    var account = _C(exchange.GetAccount());
    
    // Calculate the amount of Bitcoin that can be bought.
    var amountBuy = _N((account.Balance / buyPrice - 0.1), 2);
    // Calculate the amount of Bitcoin that can be sold, given no position limit and limited capital.
    var amountSell = _N(account.Stocks, 2);

    if (amountSell > 0.02) {
        exchange.Sell(sellPrice, amountSell);
    }
    if (amountBuy > 0.02) {
        exchange.Buy(buyPrice, amountBuy);
    }

    // Sleep before the next tick.
    Sleep(sleeptime);
}
```
